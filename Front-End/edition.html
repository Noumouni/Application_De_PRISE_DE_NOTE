Architecture microservices pour le M

Ce document présente nos réflexions sur la décomposition de l'application en services indépendants. L'objectif est de garantir une scalabilité horizontale dès le lancement.

Services principaux

API Gateway — point d'entrée unique, authentification JWT, rate limiting
User Service — gestion des comptes, profils et workspaces
Note Service — CRUD des notes, versioning, recherche full-text
Notification Service — événements in-app, e-mails, mentions

Communication inter-services

Les services communiquent via événements asynchrones pour les opérations non-critiques, et en appels REST synchrones pour les requêtes nécessitant une réponse immédiate.
